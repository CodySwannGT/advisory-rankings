# AdvisorBook async surface matrix

This matrix tracks browser-visible AdvisorBook routes that issue async
resource requests. It is the handoff surface for loading, error, retry,
desktop evidence, and mobile evidence coverage.

## Scope

- Source routes were inspected from `src/web/*.ts`.
- Legacy checked-in directory pages were inspected from `harper-app/web/*.html`.
- Shared navbar/session/search behavior applies anywhere a page mounts one of
  the design-system page templates with `refreshMe` and `search`.
- Generated deploy JavaScript under `harper-app/web/**/*.js` is ignored by git
  and is not the source of record; regenerate it with `bun run build`.

## Hidden and legacy surface classification

| Surface | Classification | Rationale |
|---|---|---|
| Legacy query-string profile URLs (`/advisor.html?id=...`, `/firm.html?id=...`, `/team.html?id=...`, `/article.html?id=...`) | Duplicate coverage required with canonical routes. | `src/web/urls.ts` intentionally supports legacy query IDs and canonicalizes successful renders to clean slug routes. Use the same profile assertions with one query-string smoke per entity kind. |
| Generated deploy JavaScript in `harper-app/web/**/*.js` | Not a separate coverage target. | The generated files are build output from `src/web/` and are ignored by git. Coverage should target source behavior, then run `bun run build` to prove generated assets are current. |
| Checked-in legacy directory HTML (`advisors.html`, `firms.html`, `teams.html`, `regulatory.html`) | Follow-up migration candidate, but currently in scope. | These pages are browser-reachable and issue public resource requests, so they need coverage now. A future migration can move them under `src/web/` and keep the same matrix rows. |
| Static-only pages without resource calls | Out of scope for async matrix. | `harper-app/web/*.html` pages that do not call `api()`, `postJson()`, `refreshMe()`, or `search()` have no route-specific async surface beyond shared template behavior. |
